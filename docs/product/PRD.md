## 1. 产品概述

### 1.1 产品背景

企业日常运营中产生大量的费用报销需求，传统的纸质报销流程存在以下问题：

- **流程繁琐**: 需要填写纸质表单，逐级审批，周期长
- **信息不透明**: 报销进度难以实时追踪，员工体验差
- **数据分散**: 报销数据难以统计和分析，不利于成本控制
- **审批效率低**: 审批人无法及时处理，导致报销延迟
- **合规风险**: 缺乏系统化的审批规则，容易出现合规问题

### 1.2 产品定位

ClaimFlow 是一款面向中大型企业的智能化费用报销管理系统，通过数字化工作流引擎实现报销流程的自动化、透明化和智能化管理。

### 1.3 核心价值

- **提升效率**: 将报销周期从平均 7-10 天缩短至 2-3 天
- **降低成本**: 减少人工处理成本 60% 以上
- **增强透明度**: 实时追踪报销状态，提升员工满意度
- **强化合规**: 自动化审批规则，确保财务合规
- **数据驱动**: 提供全面的报销数据分析，支持决策

### 5.2 核心功能详细说明

#### 5.2.1 报销申请功能

**功能描述**: 员工可以创建、编辑、提交费用报销申请

**业务流程**:
```
1. 员工登录系统
2. 进入报销申请页面
3. 填写报销信息
   - 报销类型
   - 报销金额
   - 报销日期
   - 报销说明
   - 上传附件
4. 保存为草稿 或 直接提交
5. 提交后进入审批流程
```

**字段说明**:

| 字段名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| 报销单号 | String | 自动 | 系统自动生成 | CLM-0001 |
| 报销类型 | Enum | 是 | 差旅/餐饮/交通/其他 | Travel |
| 报销金额 | Decimal | 是 | 报销总金额 | 1250.00 |
| 币种 | Enum | 是 | 货币类型 | USD |
| 报销日期 | Date | 是 | 费用发生日期 | 2025-03-08 |
| 报销说明 | Text | 是 | 详细说明 | 出差北京客户会议 |
| 附件 | File | 否 | 发票/收据 | receipt.pdf |
| 状态 | Enum | 自动 | 当前状态 | Pending |

**业务规则**:
- 报销金额必须大于 0
- 报销日期不能晚于当前日期
- 单笔报销金额超过 5000 元需要额外审批
- 附件大小不超过 10MB
- 支持的附件格式: PDF, JPG, PNG

#### 5.2.2 审批管理功能

**功能描述**: 审批人可以查看、审批、拒绝报销申请

**审批流程**:
```
1. 审批人登录系统
2. 查看待审批列表
3. 选择报销申请
4. 查看申请详情
5. 执行审批操作
   - 批准: 流转到下一审批人
   - 拒绝: 退回给申请人
   - 委托: 转给其他审批人
6. 添加审批意见（可选）
7. 提交审批结果
```

**审批规则**:

| 金额范围 | 审批流程 |
|---------|---------|
| ≤ ¥1,000 | 直属经理 → 财务 |
| ¥1,001 - ¥5,000 | 直属经理 → 财务专员 → 财务主管 |
| > ¥5,000 | 直属经理 → 财务专员 → 财务主管 → 财务总监 |

**权限矩阵**:

| 角色 | 查看权限 | 审批权限 | 管理权限 |
|------|---------|---------|---------|
| Employee | 仅本人 | 无 | 无 |
| Manager | 本部门 | 第 1 级 | 无 |
| Finance | 全部 | 第 2-3 级 | 部分管理 |
| Admin | 全部 | 全部 | 全部管理 |

#### 5.2.3 工作流引擎功能

**功能描述**: 支持自定义审批工作流，实现灵活的审批流程配置

**节点类型**:

| 节点类型 | 图标 | 说明 | 配置项 |
|---------|------|------|--------|
| 开始节点 | ○ | 流程起点 | 无 |
| 审批节点 | □ | 需要审批 | 审批人、审批规则 |
| 条件节点 | ◇ | 条件判断 | 条件表达式 |
| 动作节点 | ▱ | 自动执行 | 执行动作 |
| 结束节点 | ● | 流程终点 | 无 |

**条件类型**:
- 金额条件: amount > 1000
- 部门条件: department == 'Finance'
- 角色条件: role == 'Manager'
- 组合条件: amount > 1000 && department == 'Sales'

**工作流示例**:
```
开始 → 经理审批 → [金额判断]
                      ↓
                金额 > 5000?
                 /        \
               是          否
                ↓           ↓
          财务总监审批   财务审批
                \           /
                 \         /
                  \       /
                   结束节点
```

## 7. 业务规则

### 7.1 报销业务规则

#### BR-001: 报销金额限制

**规则描述**: 不同类型报销有不同金额限制  
**规则表达式**:
```
IF 报销类型 == '差旅' THEN 最大金额 = 50000
IF 报销类型 == '餐饮' THEN 最大金额 = 5000
IF 报销类型 == '交通' THEN 最大金额 = 2000
```

#### BR-002: 报销时效限制

**规则描述**: 报销申请必须在费用发生后 30 天内提交  
**规则表达式**:
```
当前日期 - 费用发生日期 ≤ 30 天
```

#### BR-003: 重复报销检查

**规则描述**: 系统自动检查是否存在重复报销  
**检查逻辑**:
- 相同申请人
- 相同金额
- 相同日期
- 相同类型

### 7.2 审批业务规则

#### BR-004: 审批时效

**规则描述**: 审批人必须在 3 个工作日内完成审批  
**超时处理**: 自动提醒审批人，并抄送给上级

#### BR-005: 审批人缺席

**规则描述**: 审批人连续 5 个工作日未登录系统  
**处理方式**: 自动委托给其上级或指定的代理人

#### BR-006: 审批金额权限

**规则描述**: 不同角色有不同的审批金额权限  
**权限矩阵**:
```
Manager: ≤ ¥5,000
Finance: ≤ ¥20,000
Finance Lead: ≤ ¥100,000
Admin: 无限制
```

### 7.3 工作流业务规则

#### BR-007: 工作流版本管理

**规则描述**: 工作流修改后生成新版本  
**版本策略**:
- 正在执行的流程使用旧版本
- 新发起的流程使用新版本
- 保留历史版本记录

#### BR-008: 紧急流程处理

**规则描述**: 紧急情况下可以跳过某些审批节点  
**跳过条件**:
- 需要 Admin 权限
- 需要填写跳过原因
- 记录在审计日志中
