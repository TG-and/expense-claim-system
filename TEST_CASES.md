# 费用报销系统测试用例设计

## 项目概述
- **项目名称**: 企业费用报销申请系统 (Expense Claim System)
- **技术栈**: React 19 + TypeScript + Express.js + SQLite
- **测试类型**: 单元测试 + 集成测试 + E2E测试
- **测试框架**: Jest + Supertest (后端) + Playwright (E2E)

---

## 二、E2E测试用例

### 场景1: 员工创建报销单并审批流程
| 用例ID | 测试步骤 | 预期结果 |
|--------|----------|----------|

| E2E-002-经理审批 | 1. 经理登录<br>2. 进入审批页面<br>3. 审批通过 | 状态变为Pending Finance |
